Laravel的异常处理

Published on Feb 28, 2018 ,1 min reading time

Php Laravel

在进行项目操作时，

// class A文件
public function a()
{
  try{
      logger(111);
      (new \App\Business\B)->b();
  }catch(BusinessException $e) {
      logger(444)
      $this->rollBackOrder();
      throw new BusinessException($e->getMessage().'请联系管理员处理', 21000);
  }
}

public function rollBackOrder()
{
  // some code
}

// class B文件
public function b()
{
  try{
    $i = 1;
    if ($i != 0) {
      logger(222)
      throw new KrException('错误了，回滚回滚', 500);
    }
  }catch (KrException $e) {
    logger(333)
    throw new KrException($e->getMessage(), 500);
  }
}

以上代码打印结果为:

111
222
333

可以看到最后A内的rollBackOrder方法未执行

然而把B文件改为

public function b()
{
  try{
    $i = 1;
    if ($i != 0) {
      logger(222)
      throw new BusinessException('错误了，回滚回滚', 500);
    }
  }catch (BusinessException $e) {
    logger(333)
    throw new BusinessException($e->getMessage(), 500);
  }
}

使用同一个Exception，运行结果

111
222
333
444

所以，同一个异常内，下层异常可以抛回到上层；但是不同的异常类，不能够抛回。
